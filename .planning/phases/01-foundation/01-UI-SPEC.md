# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase 1 scope note:** This phase establishes the foundational design tokens and dark theme implementation
> pattern. The App.jsx output is a placeholder shell — no user-facing UI elements are delivered in Phase 1.
> All tokens defined here are inherited by Phases 2 (sidebar), 3 (chat view), and 4 (settings panel).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Tailwind CSS v4 utility classes only (source: CONTEXT.md D-04) |
| Preset | not applicable |
| Component library | none — utility-first only, no component library (source: CONTEXT.md D-04) |
| Icon library | none in Phase 1 — deferred to Phase 2 when sidebar icons are needed |
| Font | system-ui stack (no custom font loading in Phase 1; avoids network requests at startup) |

**Tailwind v4 dark mode implementation (source: RESEARCH.md Pattern 3):**
- Dark mode strategy: class-based via `@custom-variant dark (&:where(.dark, .dark *))`
- Applied at: `<html class="dark">` in `client/index.html`
- Dark is the only theme — no light mode toggle in v1
- CSS entry point: `client/src/index.css` with `@import "tailwindcss"` + `@custom-variant dark` declaration

## Spacing Scale

Declared values (multiples of 4 only, 8-point scale):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, tight label spacing |
| sm | 8px | Compact element spacing, button inner padding |
| md | 16px | Default element spacing, card padding, input padding |
| lg | 24px | Section padding, sidebar item height |
| xl | 32px | Layout gaps, content area padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Tailwind class mapping:**

| Token | Tailwind class |
|-------|---------------|
| xs | `p-1`, `gap-1`, `m-1` |
| sm | `p-2`, `gap-2`, `m-2` |
| md | `p-4`, `gap-4`, `m-4` |
| lg | `p-6`, `gap-6`, `m-6` |
| xl | `p-8`, `gap-8`, `m-8` |
| 2xl | `p-12`, `gap-12`, `m-12` |
| 3xl | `p-16`, `gap-16`, `m-16` |

Exceptions: none

## Typography

| Role | Size | Weight | Line Height | Tailwind classes |
|------|------|--------|-------------|-----------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm font-normal leading-normal` |
| Label | 12px | 500 (medium) | 1.25 | `text-xs font-medium leading-tight` |
| Heading | 16px | 600 (semibold) | 1.25 | `text-base font-semibold leading-tight` |
| Display | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold` |

**Font family:** `font-sans` (maps to system-ui, -apple-system, Segoe UI, Roboto — no web font loaded)

**Weights used:** regular (400) and semibold (600). Medium (500) used only for Label role (tight list items).

## Color

Dark theme palette matching Claude/ChatGPT dark mode aesthetic.

| Role | Tailwind class | Hex (approx) | Usage |
|------|---------------|--------------|-------|
| Dominant (60%) | `bg-gray-950` | #0a0a0a | Main background, full-page root surface |
| Secondary (30%) | `bg-gray-900` | #171717 | Sidebar background, cards, input containers, nav rail |
| Surface-raised | `bg-gray-800` | #262626 | Hover states, active items, input fields |
| Accent (10%) | `text-blue-400` / `bg-blue-500` | #60a5fa / #3b82f6 | Primary CTA button background, active session indicator (Phase 2), send button (Phase 3) |
| Destructive | `text-red-400` / `bg-red-600` | #f87171 / #dc2626 | Delete session confirmation (Phase 2 only) |

**Accent reserved for (explicit list):**
1. Primary action button background (send message — Phase 3)
2. Active/selected session indicator in sidebar (Phase 2)
3. Focus ring on interactive elements (`ring-blue-500`)
4. Settings save button (Phase 4)

Accent is NOT used for: headings, body text, borders, backgrounds, icons in idle state.

**Text color system:**

| Role | Tailwind class | Usage |
|------|---------------|-------|
| Primary text | `text-gray-100` | Body copy, headings |
| Secondary text | `text-gray-400` | Timestamps, labels, placeholder text |
| Muted text | `text-gray-600` | Disabled states, hint text |
| Link/accent text | `text-blue-400` | Inline links only |

**Border:**
- Default: `border-gray-800` (subtle, low contrast — consistent with ChatGPT dark borders)
- Focused input: `border-gray-600`

**App shell base (applied in Phase 1):**
```
<html class="dark">
  <body class="bg-gray-950 text-gray-100 font-sans antialiased">
```

## App Shell Structure

Phase 1 delivers a minimal placeholder shell. Subsequent phases fill each region.

```
┌─────────────────────────────────────────────┐
│  <html class="dark">                        │
│  <body class="bg-gray-950 text-gray-100">   │
│                                             │
│  ┌─────────────────────────────────────┐   │
│  │  <div id="root">                   │   │
│  │  <div class="min-h-screen flex">   │   │
│  │                                     │   │
│  │   [sidebar slot — Phase 2]          │   │
│  │   [main content slot — Phase 2/3]   │   │
│  │                                     │   │
│  │  Phase 1: renders placeholder only  │   │
│  └─────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

**Phase 1 App.jsx output (placeholder only):**
```jsx
// Intentionally minimal. Sidebar and chat panel added in Phase 2+.
<div className="min-h-screen flex items-center justify-center bg-gray-950 text-gray-100">
  <p className="text-gray-400 text-sm">MMO Agent Hub — loading...</p>
</div>
```

**Layout pattern for Phase 2+ to inherit:**
```
flex h-screen overflow-hidden
  ├── sidebar: w-64 shrink-0 bg-gray-900 border-r border-gray-800  (Phase 2)
  └── main:    flex-1 flex flex-col overflow-hidden                  (Phase 2)
       ├── chat-view: flex-1 overflow-y-auto                         (Phase 3)
       └── input-bar: shrink-0 border-t border-gray-800             (Phase 3)
```

This layout contract is established in Phase 1 even though only the outer shell renders. It prevents Phase 2 from having to restructure the root DOM.

## Interaction Contracts

Phase 1 has no interactive elements. These contracts are established here for Phase 2+ inheritance.

| Interaction | Behavior | Tailwind pattern |
|-------------|----------|-----------------|
| Button hover | Background lightens one step | `hover:bg-gray-700` (secondary), `hover:bg-blue-600` (accent) |
| Button active | Slight scale compression | `active:scale-95 transition-transform` |
| Focus ring | Visible keyboard focus on all interactive elements | `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-blue-500 focus-visible:ring-offset-2 focus-visible:ring-offset-gray-950` |
| Input focus | Border highlight | `focus:border-gray-600 focus:ring-1 focus:ring-blue-500` |
| Transition speed | Fast (150ms) for hovers; standard (200ms) for opens | `transition-colors duration-150` |
| Disabled state | Reduced opacity, no pointer | `disabled:opacity-40 disabled:cursor-not-allowed` |
